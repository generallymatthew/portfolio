Font Design and Cryptography
Educational Initiative
Overview

The Telegraphic Grid Font Project bridges analog design, data logic, and history by challenging teams to collaborate on creating, encoding, and decoding a custom typeface using a highly constrained physical communication system. Over 5 days (1–1.5 hours per day), students design a custom alphanumeric font on a shared 5×5 grid system, translate visual designs into binary sequences and decimal codes, and use physical tactile signals (taps and scratches) to transmit their encryption.

The project progresses through the following phases:

  • Day 1: Design & Grid Setup — Establish the design framework and brainstorm the font on a 5×5 grid system.
  • Day 2: Character Development — Finalize the visual alphabet with complete stylistic consistency.
  • Day 3: Binary Encoding & Decimal Shorthand — Translate visual design into data logic using binary sequences.
  • Day 4: Message Composition & Poster Layout — Prep the physical canvas and establish the control message (haiku).
  • Day 5: Transmission, Inking & Verification — Run the communication loop, complete the artwork, and verify accuracy.
Design & Grid Setup
Day 1

Objective: Establish the design framework and brainstorm the font.

Instructions:
  • Introduce the project theme, constraints (binary/Telegram), and final goals to the team.
  • Review the technical constraints of a 5×5 grid system.
  • Distribute the Coder's Workbook and set up individual grid templates for sketching.
  • Begin brainstorming and sketching initial modular shapes for letters A–Z.
Material Qty
Poster Board (22×28") 1 sheet
Sharpies (Various Sizes) Set
T-Square 1 per team
Pencil 1 per student
Eraser 1 per student
Grid Paper 1 pad
Character Development
Day 2

Objective: Finalize the visual alphabet with complete stylistic consistency.

Instructions:
  • Complete the technical drawings for the full alphabet set on clean workbook grid templates.
  • Review all characters as a team to ensure stylistic uniformity across the entire set.
  • Refine any messy grid patterns to optimize clarity and legibility before translation.
Binary Encoding & Decimal Shorthand
Day 3

Objective: Translate visual design into data logic.

Instructions:
  • Conduct a final review of the visual alphabet.
  • Translate each letter row-by-row into a 5-digit binary sequence (0 for empty space, 1 for filled space).
  • Map these binary sequences into their specific decimal shorthand values using the ledger in the workbook.
  • Document the completed system codes cleanly so they are ready for transmission.

HAIKU RULES

  • Structure: Three lines with a 5-7-5 syllable pattern
  • Simplicity: Uses clear, concrete imagery rather than abstract ideas
  • Present tense: Describes what is happening now, creating immediacy
Message Composition & Poster Layout
Day 4

Objective: Prep the physical canvas and establish the control message.

Instructions:
  • Lock in the pre-decided haiku or short phrase that will serve as your translation test.
  • Plan the layout, letter spacing, and margins on the large poster sheet.
  • Gently sketch out the master layout grid boundaries on the poster sheet using pencil and ruler.
Transmission, Inking & Verification
Day 5

Objective: Run the communication loop, complete the artwork, and verify accuracy.

Instructions:
  • Transmit: Senders transmit the decimal/binary data using rhythmic taps (dots/zeros) and scratches (dashes/ones).
  • Transcribe: Receivers log the codes, map them back into the 5×5 poster grids, and lightly pencil them in.
  • Ink: Color in the filled squares cleanly using solid black fine-line markers and thick permanent markers.
  • Verify (QA): Read the final visual poster layout side-by-side with the original control haiku to check for discrepancies and patch any translation errors.
